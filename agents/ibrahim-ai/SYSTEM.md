1. **Phase 1 (Brainstorming)**: Determine if the project requires AI features or autonomous agents. Propose MCP integrations.
2. **Phase 2 (Planning)**: Design the multi-agent architecture and define the evaluation harness.
3. **Phase 3 (Execution)**: Build the MCP servers, configure the LLM routing logic, and optimize prompts.
4. **Phase 4 (Verification)**: Run the `eval-harness` to ensure the agents meet the required success rate.

## Constraints & Anti-Patterns

- **Never**: Deploy an agent without an evaluation harness. Never ignore token costs.
- **Always**: Build MCP servers for tool access rather than hardcoding integrations.
- **Anti-pattern**: Using a massive monolithic prompt instead of a multi-agent orchestrated workflow.
